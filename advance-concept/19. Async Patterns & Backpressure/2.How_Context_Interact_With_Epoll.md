# Big Picture

```go
ctx, cancel := context.WithTimeout(..., 3s)

req, _ := http.NewRequestWithContext(ctx, ...)
client.Do(req)
```

Underneath:

```
goroutine
  ↓
net/http
  ↓
net package
  ↓
nonblocking socket fd
  ↓
Go runtime netpoller
  ↓
epoll (Linux) / kqueue (BSD/macOS)
```

# Normal Network Read Without Context

Suppose goroutine waits for response:

```
conn.Read(buf)
```

Because socket is non-blocking:

Kernel returns:

```
EAGAIN
```

Meaning:

```
no data now
```

Go runtime does NOT block thread.

Instead:

```
goroutine parks
fd registered in epoll
M thread free to run others
```

# Runtime Structures

Internally:

```
FD -> pollDesc
pollDesc stores:
- read waiter goroutine
- write waiter goroutine
- deadlines
- seq state
```

Runtime waits in:

```
epoll_wait()
```

or

```
kevent()
```

# Where Context Enters

### If request has timeout:

```
ctx, cancel := context.WithTimeout(...3s)
```

## There are two parallel waiting systems:

### A. Network readiness

epoll says socket readable

### B. Context cancellation

timer fires OR caller cancel()

```
Whichever happens first wins.
```

# Detailed Timeline Example

### Step 1

Goroutine sends HTTP request.

Then waits for response body.

```
G1 waiting read(fd=42)
```

Runtime parks G1.

Thread continues others.

### Step 2

Socket silent.

No response yet.

Kernel epoll still waiting.

### Step 3

3 seconds timeout fires.

Context timer closes:

```
ctx.Done()
```

Now net/http notices cancellation.

### Step 4

HTTP transport forcibly aborts socket operation.

Often does one of:

```
close(fd)
shutdown(fd)
set deadline immediate
wake poller
```

### Step 5

epoll_wait wakes because fd changed.

Runtime marks goroutine runnable.

G1 resumes.

Read returns error:

```
context deadline exceeded
```

or wrapped timeout error.

# Finaly

```
Context = lifecycle control for work

epoll = waits for socket readiness

Go runtime netpoller = bridge between kernel events and goroutines

When context cancels:
    higher-level code closes connection / sets deadlines
    epoll wakes
    goroutine resumes
    returns canceled error
```
