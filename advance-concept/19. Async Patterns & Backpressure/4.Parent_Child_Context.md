# Context cancellation propagates downward to children.

```
parent canceled -> all children canceled
child canceled -> siblings unaffected
```

## Example: Shared Parent (Cancels All Children)

```go
root := context.Background()
ctx, cancel := context.WithTimeout(root, 5*time.Second)

go taskA(ctx)
go taskB(ctx)
go taskC(ctx)
```

After 5 sec:

```
taskA canceled
taskB canceled
taskC canceled
```

Because all share same ctx.

## How to Cancel Only Specific Function

```go
root := context.Background()

ctxA, cancelA := context.WithCancel(root)
ctxB, cancelB := context.WithCancel(root)
ctxC, cancelC := context.WithCancel(root)

go taskA(ctxA)
go taskB(ctxB)
go taskC(ctxC)
```

Now:

```
cancelB()
```

Results:

```
taskB canceled
taskA keeps running
taskC keeps running
```

# Important Rule

If parent request context is canceled:

```
reqCtx canceled
↓
profileCtx canceled
recoCtx canceled
notifCtx canceled
```

Parent always controls children.

# If You Want Isolation from Parent Too

Use new root:

```go
independentCtx, cancel := context.WithTimeout(context.Background(), 5*time.Second)
```

Now it won’t cancel when request context cancels.

Use carefully.

# MOdel

| Function                                | Purpose                                                       | Use Case                                                      |
| --------------------------------------- | ------------------------------------------------------------- | ------------------------------------------------------------- |
| `context.Background()`                  | Returns an empty, non-cancelable root context.                | Top-level of an app (`main`, initialization, tests setup).    |
| `context.WithCancel(parent)`            | Creates a child context that can be canceled manually.        | Stop goroutines, worker pools, background jobs gracefully.    |
| `context.WithTimeout(parent, duration)` | Creates a child context that auto-cancels after a timeout.    | HTTP requests, DB queries, RPC calls with max execution time. |
| `context.WithDeadline(parent, time)`    | Creates a child context that auto-cancels at a specific time. | Tasks that must finish before an exact deadline.              |
| `context.WithValue(parent, key, val)`   | Attaches request-scoped metadata to a context.                | Request IDs, trace IDs, auth metadata (sparingly).            |
