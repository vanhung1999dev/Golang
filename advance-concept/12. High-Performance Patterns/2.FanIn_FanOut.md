# 1. Core Definitions
## Fan-Out

One input source distributes work to many workers.

```
          jobs
            |
    -----------------
    |   |   |   |   |
   W1  W2  W3  W4  W5
```

Used to parallelize work.

## Fan-In

Many producers combine outputs into one stream.

```
W1 \
W2  \
W3   ---> results channel
W4  /
W5 /
```

Used to aggregate results.

## Combined Pattern

Most systems use both:

```
Input -> Fan-Out workers -> Fan-In results -> consumer
```

# Real Go Example
```go
jobs := make(chan int)
results := make(chan int)

for i := 0; i < 4; i++ {
    go worker(jobs, results)
}
```

- jobs fan-out to workers
- results fan-in from workers

# Fan-Out Deep Dive
Example
```go
for i := 0; i < 3; i++ {
    go worker(jobs)
}
```

All workers read same channel:

```go
func worker(jobs <-chan Job) {
    for job := range jobs {
        process(job)
    }
}
```

## How Runtime Behaves

Workers block on:

```go
job := <-jobs
```

When producer sends:

```go
jobs <- task
```

- **Runtime wakes one waiting receiver**.

- **Not all.**

That means channel naturally load-balances.

## Example

3 workers waiting:

```
recvq = [W1 W2 W3]
```

Send task A:

```
W1 wakes -> gets A
```

Send task B:

```
W2 wakes -> gets B
```

Send task C:

```
W3 wakes -> gets C
```

This creates natural fan-out.

## Important Truth

Fan-out via shared channel means:

```
each task consumed once
one worker only
```

Not broadcast.

# Fan-In Deep Dive
Workers write to same output channel:

```
results <- value
```

Many producers, one consumer.

## Example
```go
func worker(jobs <-chan int, out chan<- int) {
    for n := range jobs {
        out <- n * 2
    }
}
```

Consumer:

```go
for v := range results {
    fmt.Println(v)
}
```

## Internal Behavior

- Many workers may attempt send simultaneously.

- Channel serializes access via lock + wait queues.

- So fan-in can become contention point.

# Ordering Tradeoff

#### Fan-out breaks deterministic order.

Input:

```
1 2 3 4 5
```

Output may become:

```
2 6 4 10 8
```

**Because faster workers finish earlier**

# If Order Required

Need:
```
sequence numbers
reorder buffer
single-threaded merge
```
