# 1.What Worker Pool Really Is
Worker pool means:

```
Create fixed number of goroutines once
Reuse them to process many jobs
```

Instead of:

```
1 request = create 1 new goroutine
```

we do:

```
1000 requests
10 workers handle them gradually
```

# 2. Basic Code
```go
jobs := make(chan Job, 100)

for i := 0; i < 5; i++ {
    go worker(jobs)
}
```

Worker:

```go
func worker(jobs <-chan Job) {
    for job := range jobs {
        process(job)
    }
}
```

# 3. Why range jobs Looks Forever

This line:

```go
for job := range jobs
```

means:

```
Keep receiving values from channel
Until channel is closed
```

Equivalent to:

```go
for {
    job, ok := <-jobs
    if !ok {
        break
    }

    process(job)
}
```

So yes, **it can run "forever" if channel never closes.**

That is normal in servers.

# Your Biggest Confusion: Does Goroutine End After One Job?
Answer:**NO**

Worker goroutine does:

```
receive job1
process job1
receive job2
process job2
receive job3
...
```

Same goroutine reused many times.

**It does not die after one task.**

# 5. Then What Happens After One Job Finishes?

Example:

```go
for job := range jobs {
    process(job)
}
```

After process(job1) ends:

worker goes back to top of loop:

```go
job := <-jobs
```

Then waits for next job.

So lifecycle:

```
wake up
do work
wait
wake up
do work
wait
```

# 6. What If No Data In Channel?

Worker reaches:

```go
job := <-jobs
```

No data available.

Then runtime parks goroutine.

Meaning:

```
paused
sleeping
not using CPU
still alive
```

It waits inside channel receive queue.


Producer sends:
```
jobs <- job
```

Runtime wakes one waiting worker.

Then worker resumes exactly after receive line:

```
job := <-jobs
```

and processes it.

# 8. When Is Goroutine Actually Released / Destroyed?

Only when worker function returns.

Worker function:

```go
func worker(jobs <-chan Job) {
    for job := range jobs {
        process(job)
    }
}
```

This returns only when channel closes:

```
close(jobs)
```

Then:

```
channel has no more jobs
range loop ends
function returns
goroutine dies
memory reclaimed later
```
