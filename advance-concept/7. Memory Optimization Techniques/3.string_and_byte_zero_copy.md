# Everything Is Bytes in Memory

Text like:

```
"hello"
```

is stored as bytes:

```
h e l l o
104 101 108 108 111
```

So both string and []byte ultimately refer to bytes.

**Difference is how Go wraps and manages those bytes.**

# string Internal Layout

Conceptually:

```go
type string struct {
    ptr *byte
    len int
}
```

**(Real runtime uses internal structs, but this mental model is correct.)**

So string contains:

```
pointer to byte data
length
```

Example:

```
s := "hello"
```

Memory concept:

```
s:
ptr ---> [h][e][l][l][o]
len = 5
```

## Important: String Is Immutable

You cannot do:

```
s[0] = 'H'
```

Compile error.

```
Because Go guarantees:

string data never changes
```

This allows:

```
safe sharing
optimization
hash/map stability
no accidental mutation
```

# []byte Internal Layout

Conceptually:

```go
type slice struct {
    ptr *byte
    len int
    cap int
}
```

So byte slice has:

- pointer to backing array
- current length
- capacity

Example:

```go
b := []byte{'h','e','l','l','o'}
```

Memory:

```
b:
ptr ---> [h][e][l][l][o][...free...]
len = 5
cap = maybe 5+
```

## Why Slice Has Capacity

Because slices grow.

```
b = append(b, '!')
```

If cap allows:

```
reuse same backing array
```

If full:

```
allocate larger array
copy old bytes
move ptr
```

# String vs []byte Summary

string:
```
ptr + len
read-only
```

[]byte:
```
ptr + len + cap
```
- mutable
- growable via append

# Why They Are Separate Types

If Go allowed free shared conversion:

string <-> []byte same memory

Then mutable bytes could silently mutate immutable strings.

Bad.

So Go separates semantics.

# Conversion string -> []byte
```go
s := "hello"
b := []byte(s)
```

Usually does:

```
1. allocate new byte array len=5
2. copy h e l l o
3. return slice header
```

Why copy?

```
Because bytes are mutable.
```

If shared memory:

```
b[0] = 'H'
```

would mutate string.

Not allowed.

# Conversion []byte -> string
```go
b := []byte("hello")
s := string(b)
```

Usually does:

```
1. allocate new string bytes
2. copy data
3. return string header
```

Why?

Because later:
```
b[0] = 'H'
```

must not affect string.

# Why Conversions Cost

Each conversion may involve:

```
allocation
memory copy
GC pressure
```

At scale:

```
50KB body × 20k req/s
```

Huge traffic.

# Zero-Copy Slicing with []byte

```go
data := []byte("hello world")
part := data[:5]
```

**Usually no copy.**

**Both share same backing array.**

```
data ptr ----+
part ptr ----+
```

# String Slicing
```
s := "hello world"
sub := s[:5]
```

**Usually no copy too.**

**New string header points into same bytes.**

```
sub -> "hello"
```

Still immutable.

# Why Use unsafe Here?

To avoid conversion copy:

```
[]byte -> string
string -> []byte
```

by reusing same memory.

# Unsafe []byte -> string

Modern concept:

```
b := []byte("hello")
s := unsafe.String(&b[0], len(b))
```

Now:

```
No copy
s points to b memory
```
