# Why sync.Pool Exists
In Go, creating objects repeatedly costs:

```
malloc allocation
heap growth
GC scanning
GC cleanup
CPU time
```

Example:

```go
func handler() {
    buf := make([]byte, 0, 4096)
}
```

If 100k requests/sec:

100k allocations/sec

Even if object is temporary.

So Go provides:

```go
sync.Pool
```

to reuse temporary objects.

# What sync.Pool Is

Think:

```
Reusable object shelf
```

Instead of:

```
Create object → use once → garbage
```

We do:

```
Take object from shelf
Use it
Clean it
Return to shelf
Next request reuses it
```

# Core API

```go
var pool sync.Pool
```

Main methods:

```go
pool.Get()
pool.Put(obj)
```

Optional constructor:

```
New: func() any { ... }
```

# Full Basic Syntax
```go
var pool = sync.Pool{
    New: func() any {
        return make([]byte, 0, 4096)
    },
}
```

Meaning:

If pool empty:

```
create []byte buffer cap 4096
```

# How Get() Works
```go
buf := pool.Get().([]byte)
```

Means:

```
Try reuse old object
If none exists -> call New()
Return object to caller
```

# How Put() Works
```go
pool.Put(buf)
```

Means:

```
Return object to pool storage
May be reused later
```

# Why Reset() Matters

Suppose:

```go
buf := pool.Get().([]byte)
buf = append(buf, "secret"...)
pool.Put(buf)
```

Next user may get old content.

So clean first:

```go
buf = buf[:0]
pool.Put(buf)
```

For bytes.Buffer:

```go
b.Reset()
pool.Put(b)
```

# The Exact Confusing Part You Had: Ownership
Question:
```
If goroutine A gets object X,
why goroutine B cannot get same X simultaneously?
```
Real Answer:

```
Because Get() REMOVES object X from pool storage.
```

If shelf has:

```
[A][B][C]
````

Goroutine A:

```
x := pool.Get()
```

Gets A.

Shelf now:

```
[B][C]
```

A is no longer available.

So another goroutine cannot get A.

# sync.Pool Uses Per-P Local Storage

Instead of one global shelf:

```
Pool:
P0 local shelf
P1 local shelf
P2 local shelf
P3 local shelf
```

So goroutine on P2 usually uses P2 shelf first.

Less contention.

### Internal Structure (Simplified)

Each P has:
```
private slot   (super fast)
shared queue   (extra items)
```

# Get() Internal Flow

Goroutine running on P2:

```go
pool.Get()
```

Runtime roughly:

```
1. Check P2 private slot
2. Else check P2 shared queue
3. Else steal from other P shelves
4. Else call New()
```

# Put() Internal Flow

```
pool.Put(obj)
```

Runtime roughly:

```
1. If local private empty -> store there
2. Else push to local shared queue
```
