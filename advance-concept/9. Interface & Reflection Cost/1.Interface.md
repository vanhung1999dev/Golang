# 1. Interface = (Type, Value)
In Go, an interface is not magic.

Conceptually:

```go
var x interface{} = 42
```

Stores:

```
(type = int, value = 42)
```

So interface is:

```
interface = (dynamic concrete type metadata, data pointer/value)
```

# 2.Internal Runtime Layout

Go has two internal forms:

```
eface  = empty interface{}          // no methods
iface  = non-empty interface        // has methods
```

## Empty Interface (interface{})

Conceptually:

```
type eface struct {
    _type *rtype
    data  unsafe.Pointer
}
```

Meaning:

```
_type = metadata for int/string/struct/etc
data  = actual value storage
```

Example:

```
var x interface{} = "hello"
```

Stored:

```
_type -> string type descriptor
data  -> bytes of string header
```

## Non-empty Interface
```
type iface struct {
    tab  *itab
    data unsafe.Pointer
}
```

Where:

```
itab = relationship between concrete type + interface methods
```

Contains:

```
- concrete type info
- interface type info
- function pointers for methods
```

# Why Interface Exists

Allows polymorphism:

```
type Reader interface {
    Read([]byte) (int, error)
}
```

Then:

```
func Process(r Reader)
```

Can pass:

```
file
socket
buffer
gzip reader
network stream
mock in tests
```

This is static language dynamic behavior.
