# Interface

When you use an interface, Go runtime already stores:
```
(type, value)
```

So the runtime already knows the concrete type internally.

Example:

```
var x interface{} = 123
```

Internally:

```
type = int
value = 123
```

Go does not need reflection just to know the type.

## What Actually Happens with Interface

### Case 1: Type Assertion (No Reflection API)
```go
var x interface{} = 123

v := x.(int)
```

This uses runtime type checking, but not the reflect package.

It checks:

```
Is concrete type int?
Yes -> return value
No -> panic / false
```

Fast path compared to reflection.

### Case 2: Type Switch
```go
switch v := x.(type) {
case int:
case string:
}
```

Also runtime type checks.

Still not reflection package.

<br>

# When Reflection Happens

Reflection means you explicitly use:

```go
import "reflect"
```

Example:

```go
t := reflect.TypeOf(x)
v := reflect.ValueOf(x)
```

Now you're asking Go:

```
inspect metadata dynamically
what fields exist?
what methods?
set value?
field by name?
iterate struct?
call method dynamically?
```

# Final Correct Understanding
```
Interface = stores unknown concrete value with known runtime type metadata.

Reflection = explicit runtime inspection/manipulation of that metadata.
```
