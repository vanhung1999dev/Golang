# When calling:

```go
r.Read(buf)
```

Where r is interface.

Go cannot know direct function at compile time (sometimes it can inline if devirtualized, but often not).

Instead runtime does:

```
1. load itab
2. find method fn pointer
3. indirect call
```

Equivalent conceptually:

```
call (*itab.methods[0])(data, args...)
```

# Cost Compared to Direct Call

Direct call:

```
CALL concrete_function
```

Interface call:

```
load pointer
load fn ptr
indirect branch
CALL through register
```

Extra costs:

```
pointer chasing
branch prediction weaker
less inlining
less escape optimization
```

# Why It Matters at Scale

One interface call:
```
tiny cost
```

100 million calls/sec:

```
very real CPU burn
```

Examples:

```
serialization loops
packet parsing
hash pipelines
metrics hot path
routing middleware chains
```

# Example

```go
type Adder interface {
    Add(int, int) int
}
```

Loop:

```
for i := 0; i < 1e9; i++ {
    sum += a.Add(i, i)
}
```

May be slower than:

```
sum += concrete.Add(i, i)
```

Because direct call may inline.

# Why Inlining Matters

Direct function:

```go
func Add(a,b int) int { return a+b }
```

Compiler can replace with:

```
sum += i+i
```

No call overhead.

But interface call often blocks this.

Then:
```
call overhead remains
cannot optimize across boundary
```

# Escape Analysis + Interface
Very important.

```go
func Print(v interface{}) { ... }
```

Passing value into interface may force allocation.

Example:

```go
x := BigStruct{}
Print(x)
```

**Compiler may move x to heap if needed.**

Why?

```
Because interface stores boxed representation.
```

Result:

```
heap alloc
GC pressure
cache misses
```

# Boxing Cost

Primitive into interface:

```go
var x interface{} = 123
```

Need represent int as generic runtime object.

Sometimes stack stored, sometimes heap.

For loops:

```go
for _, v := range nums {
    arr = append(arr, interface{}(v))
}
```

Can generate many allocations depending context.
