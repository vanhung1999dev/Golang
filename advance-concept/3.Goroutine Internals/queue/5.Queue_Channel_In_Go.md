# Waiting structure in GO

Every blocking operation in Go follows this pattern:

```
1. Prepare wait object
2. Link current goroutine to wait structure
3. Mark G waiting
4. Drop CPU
5. Scheduler runs another G
6. Event occurs later
7. Wake goroutine (goready)
8. Put back to run queue
```
<br>

# CASE 1 — Channel Receive Deep Dive

#### Code
```go
x := <-ch
```

If no sender + no buffered value.

### Runtime Entry

Eventually reaches:
```
chanrecv()
```

### hchan Structure

Simplified:

```
type hchan struct {
    qcount
    dataqsiz
    buf
    recvq waitq
    sendq waitq
    lock
}
```
#### recvq / sendq

**Wait queues of blocked goroutines.**

But queue stores sudog.

#### sudog Structure
```go
type sudog struct {
    g    *g
    elem unsafe.Pointer
    next *sudog
}
```

# Step-by-Step Receive Block

### 1 . Running G1 executes <-ch

No buffered item. <br>

No sender waiting. <br>

### 2. Create sudog
```go
sg.g = G1
sg.elem = &x
```

Means receive result should copy into x later.

### 3. Enqueue into recvq

ch.recvq = [sg(G1)]

### 4. G1 parks

gopark(waitReasonChanReceive)

Now:

```
G1 status = waiting
```

Scheduler runs another goroutine.

## Later Sender Arrives

```go
ch <- 42
```
ch <- 42

Runtime:

### Finds waiting receiver in recvq

```
Pop G1’s sudog.
```

### Copy value directly
```
42 -> G1 stack variable x
```

### Wake G1
```
goready(G1)
```

### Put into run queue

```
Later G1 resumes after park.
```

<br>

# CASE 2 — Channel Send Deep Dive

```go
ch <- v
```

If no receiver + buffer full.

## Steps

### 1. Create sudog
```
sg.g = G2
sg.elem = &v
```

### 2. Push to sendq
```
ch.sendq = [G2]
```

### 3. gopark()
Waiting.

#### Later Receiver Comes

Receiver pops sender from sendq.

Copies sender value directly.

Wakes sender.

## Steps
### 1. Create timer

Associated with current G1.

### 2. Insert into per-P timer heap

Heap sorted by earliest deadline.

top = soonest timer
