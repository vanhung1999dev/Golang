# Why Initial Stack ~2KB Matters
Suppose:

```
500,000 idle goroutines
```

If each had 8MB thread stack:

```
4,000,000 MB = absurd
```

If each starts at 2KB:
```
~1GB range (rough mental model)
```

Massively more efficient.

## Internal Goroutine Stack Metadata

```Go
type g struct {
    stack struct {
        lo uintptr
        hi uintptr
    }
    stackguard0 uintptr
    stackguard1 uintptr
}
```

Meaning:

- lo = bottom of stack memory
- hi = top boundary
- stack grows downward on most archs
- guards detect overflow / preemption

## Memory Layout Concept

```
high address
-------------
stack top (hi)
local vars
func frames
more frames
-------------
stack bottom (lo)
low address
```

Usually grows downward:

```
new calls => SP moves lower
```
